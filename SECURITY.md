# Security Policy

## Reporting a Vulnerability

Do not open a public issue for security-sensitive reports.

Instead, report the issue privately through GitHub security reporting if enabled, or contact the maintainer directly.

Include:

- A clear description of the issue
- Impact
- Reproduction steps
- Affected environment
- Any suggested remediation

## Disclosure

Please allow time to investigate and patch the issue before public disclosure.

## Scope

Security reports are especially relevant for:

- Command execution boundaries
- Provider authentication handling
- Local state and persistence
- Install script behavior
- Path and filesystem handling
